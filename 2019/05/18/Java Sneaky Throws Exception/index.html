Java Sneaky Throws Exception

发表于 2019-05-18 | 分类于 软件开发

相关概念

Sneaky Throw 允许抛出检查异常，但无需在方法签名上throws或try catch该检查异常
在JVM层面不管方法是否有throws都可以抛异常，不区分检查及非检异常，检查异常是编译器的约束
Sneaky Throw 利用Java8的类型推断机制，把检查异常转为非检查异常，绕过编译器

Java8语言规范中的定义

大意是 如<T extends Throwable> 若T不能推断出一个具体的检查异常类型，则T会被推断为RuntimeException

使用场景

Sneaky Throw 可以避免检查异常的try catch样板代码，可用于不能抛出异常的地方，如lambda表达式中或Runnable类似的接口

使用方法

  1. 实现一个util方法，利用泛型类型推断，按java规范，E被推断为RuntimeException, 重新抛出异常

  2. 测试一下，把IOException转为非检查异常

注意事项

由于使用Sneaky Throw 后，就不能try catch具体检查异常进行异常处理了，需要谨慎使用
